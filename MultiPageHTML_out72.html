This variable supports the performance-schema-instrument='instrument_name=  
value' syntax, where instrument_name is the name of the instrument and value is  
either ON, TRUE, or 1for enabled instruments; OFF, FALSE, or 0for disabled; and COUN  
TED for those that are counted instead of TIMED. You can specify this option several  
times to enable or disable different instruments. The option also supports wildcards:  
performance-schema-instrument='statement/sql/select=ON'  
If multiple options are specified, the longer instrument string has  
precedence over the shorter regardless of the order.  
Enabling and Disabling Consumers  
Like instruments, consumers can be enabled or disabled by:  
• Updating the setup_consumerstable in Performance Schema  
• Using the stored procedures ps_setup_enable_consumer and ps_setup_  
disable_consumerin sysschema  
• Setting the performance-schema-consumerconfiguration parameter

but there are a few consumers whose names need more explanation, listed in
