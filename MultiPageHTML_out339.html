GCP Cloud SQL  
Cloud SQL is GCP’s managed MySQL offering. A core difference between this offer‐  
ing and AWS’s is that it runs the community server but with certain features disabled  
specifically to allow for the multitenancy and managed aspect of the product. Here  
are some of the things you cannot use with Cloud SQL even though it runs the com‐  
munity server:  
• SUPERprivilege is disabled.  
• Loading plug-ins are disabled.3  
• Some clients are also disabled, such as mysqldump and mysqlimport.  
Similar to AWS’s offering, you cannot get SSH access to the instances.  
3
Cloud SQL does offer its own solution for audit logging to support compliance needs.
