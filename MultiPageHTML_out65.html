Most of the instrument names are self-descriptive. As in the examples,  
statement/sql/select is a SELECT query, and wait/synch/mutex/innodb/auto  
inc_mutexis a mutex that InnoDB sets on the auto-increment column. There is also a  
DOCUMENTATION column in the setup_instruments table that may contain more  
details:  
mysql> SELECT * FROM performance_schema.setup_instruments  
-> WHERE DOCUMENTATION IS NOT NULL LIMIT 5, 5\G  
*************************** 1. row ***************************  
NAME: statement/sql/error  
ENABLED: YES  
TIMED: YES  
PROPERTIES:  
VOLATILITY: 0  
DOCUMENTATION: Invalid SQL queries (syntax error).  
*************************** 2. row ***************************  
NAME: statement/abstract/Query  
ENABLED: YES  
TIMED: YES  
PROPERTIES: mutable  
VOLATILITY: 0  
DOCUMENTATION: SQL query just received from the network. At this point, the  
real statement type is unknown, the type will be refined after SQL parsing.  
*************************** 3. row ***************************

ENABLED: YES  
TIMED: YES  
PROPERTIES: mutable  
VOLATILITY: 0  
DOCUMENTATION: New packet just received from the network. At this point,  
the real command type is unknown, the type will be refined after reading  
the packet header.

VOLATILITY: 0
